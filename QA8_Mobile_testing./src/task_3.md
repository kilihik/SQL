| | Создание эмулятора с Play Market | |

|| Включение режима разработчика на эмуляторе ||

### Создание и запуск симулятора в Xcode: краткая инструкция для коллеги**

| Шаг | Действие | Описание | Примечания (Обрати внимание!) |
|:---:|----------|-----------|-------------------------------|
| **0** | **Проверить доступные версии ОС** | Перед созданием убедись, что нужная версия iOS уже установлена. | **Критически важно!** Если версии нет в списке 'OS Version', её нужно скачать. Переходи к **Шагу 4.1**. |
| 1 | **Открыть менеджер устройств** | В Xcode выберите в меню: 'Window → Devices and Simulators. | **Горячие клавиши:** 'Cmd + Shift + 2' — самый быстрый способ. |
| 2 | **Перейти на вкладку симуляторов** | В открывшемся окне выберите вкладку **"Simulators"**. | Слева список симуляторов, справа — их настройки и информация. |
| 3 | **Добавить новый симулятор** | Нажмите значок **"+"** в левом нижнем углу окна. | Можно создать несколько симуляторов с разными устройствами и ОС. |
| 4 | **Настроить параметры** | Заполните поля в диалоговом окне: | |
| | **4.1. Simulator Name** | Дайте понятное имя, чтобы легко найти. | *Пример:* 'iPhone 15 Pro Max (iOS 17.2)' или 'QA_iPad_10th_iOS16'. |
| | **4.2. Device Type** | Выберите модель устройства из списка (iPhone, iPad, Apple TV, Apple Vision Pro). | Для тестирования интерфейса обязательно проверяй на разных типах (iPhone SE и iPhone Pro Max). |
| | **4.3. OS Version** | Выберите версию iOS/iPadOS. | **Если версии нет:** Нажмите 'Download more...' или открой 'Xcode → Settings → Platforms'. Скачивание может занять время. |
| 5 | **Создать симулятор** | Нажмите кнопку **"Create"**. | Новый симулятор появится в списке слева. |
| 6 | **Выбрать симулятор в проекте** | Вернись в основное окно Xcode. В панели инструментов рядом со схемой сборки (Scheme) выбери созданный симулятор из выпадающего списка. | Убедись, что выбран **симулятор**, а не 'Any iOS Device'. |
| 7 | **Запустить приложение** | Нажмите кнопку **"Run"** (треугольник) или 'Cmd + R'. | Xcode соберёт проект и запустит его в симуляторе. Первый запуск нового симулятора может быть дольше. |
| **8** | **Управление симулятором** | После запуска используй меню симулятора и горячие клавиши. | **Лайфхаки:** <br>• 'Cmd + R' — поворот. <br>• 'Cmd + S' — сделать скриншот. <br>• 'Cmd + Shift + H' — нажать "Home". <br>• 'Hardware → Touch ID/ Face ID' — эмулировать успешное/неуспешное распознавание. |

### **Что еще важно знать (для заметок):**
*   **Удаление симуляторов:** Чтобы не захламлять систему, старые ненужные симуляторы можно удалить. Для этого в 'Devices and Simulators' ('Cmd+Shift+2') выбери симулятор и нажми клавишу 'Delete' или кнопку '-'.
*   **Сброс содержимого:** Если приложение ведёт себя странно, можно сбросить симулятор до чистого состояния: 'Device and Simulators' → Выбери симулятор → 'Erase All Content and Settings...'.
*   **Файловая система:** Через меню симулятора ('Device' → 'Files and Folders') можно получить доступ к песочнице приложения для отладки, если оно работает с файлами.
*   **Производительность:** Симулятор работает на твоём Mac и использует его ресурсы. Если Mac старый или слабый, симулятор может тормозить, особенно на больших устройствах (iPad Pro, Vision Pro).
