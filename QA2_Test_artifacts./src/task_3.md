# Тест-план для сайта "Школа 21" - раздел "Корпоративное обучение"

## 1. Введение

Описание: Общее описание целей и задач тестирования раздела "Корпоративное обучение".
Содержание: Тестирование функциональности раздела для корпоративных клиентов, включая формы заявок, отображение информации об обучении, калькуляторы стоимости и интеграционные элементы.

## 2. Объем тестирования

Описание: Определение границ и ограничений тестирования.
Содержание:

- Функциональное тестирование форм подачи заявок

- Тестирование отображения информации о программах обучения

- Проверка калькулятора стоимости корпоративного обучения

- Тестирование адаптивности на разных устройствах

- Проверка интеграции с системами оплаты

## 3. Подход к тестированию

Описание: Методология и стратегия проведения тестирования.
Содержание: Комбинированный подход - ручное функциональное тестирование основных сценариев, проверка UI/UX, тестирование совместимости с основными браузерами (Chrome, Firefox, Safari) и мобильными устройствами.

## 4. Критерии начала и окончания тестирования

Описание: Условия для старта и завершения тестирования.
Содержание:
Начало: Раздел развернут на тестовом стенде, подготовлены тестовые данные
Окончание:

- Выполнены все запланированные тест-кейсы

- Критические и блокирующие дефекты исправлены

- Пройдены основные пользовательские сценарии

## 5. Тестовое окружение

Описание: Описание оборудования, ПО и инфраструктуры для тестирования.
Содержание:

- Тестовый сервер с копией продакшн-окружения

- Браузеры: Chrome 90+, Firefox 88+, Safari 14+

- Мобильные устройства: iOS 14+, Android 10+

- Скорость интернета: тестирование при разных условиях (3G, 4G, WiFi)

## 6. Риски

Описание: Потенциальные проблемы, которые могут повлиять на тестирование.
Содержание:

- Недоступность тестового окружения

- Изменение требований в процессе тестирования

- Нехватка времени для полного покрытия

- Проблемы с интеграцией сторонних сервисов

## 7. Ресурсы

Описание: Распределение человеческих и технических ресурсов.
Содержание:

- Тестировщик: 1 человек (выполнение тестов, документирование результатов)

- Технический специалист: поддержка тестового окружения

- Менеджер проекта: координация и принятие решений

## 8. Расписание

Описание: План и сроки выполнения работ по тестированию.
Содержание:

- Подготовка тестовой документации: 1 день

- Функциональное тестирование: 2 дня

- Тестирование совместимости: 1 день

- Регрессионное тестирование: 1 день

- Подготовка отчета: 0.5 дня

## 9. Критерии приемки

Описание: Условия, при которых раздел считается готовым к выпуску.
Содержание:

- Все функциональные требования реализованы

- Отсутствуют критические и блокирующие дефекты

- Основные пользовательские сценарии работают корректно

- Совместимость с основными браузерами подтверждена
