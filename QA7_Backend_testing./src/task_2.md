# Функции Postman для работы с API и автотестирования

| Функция | Что это? | Зачем нужно? | Как использовать (ключевые шаги) |
| :--- | :--- | :--- | :--- |
|  1. Настройка окружения (Environments)  | Набор переменных (`ключ-значение`) для управления конфигурацией под разные среды (dev, staging, prod). | 1.  Разделение сред:  Быстрое переключение между серверами.<br>2.  Безопасность:  Хранение секретов (токены, ключи) в переменных.<br>3.  Повторное использование:  Один запрос работает в любом окружении.<br>4.  Упрощение:  Сложные URL и данные задаются один раз. | 1.  Создать:  Слева вкладка "Environments" → "+".<br>2.  Задать переменные:  Добавить пары, напр. `base_url` = `https://api.dev.com`.<br>3.  Использовать:  В запросе писать `{{base_url}}/users`.<br>4.  Выбрать:  В выпадающем списке (правый верхний угол) выбрать активное окружение. |
|  2. Создание коллекций (Collections)  | Группа (папка) для организации связанных HTTP-запросов. Основа для автоматизации и документирования. | 1.  Организация:  Структурирование запросов (напр., "Auth", "User CRUD").<br>2.  Запуск сценариев:  Последовательный запуск всех запросов.<br>3.  Документирование:  Добавление описаний.<br>4.  Командная работа:  Общий доступ к группе запросов.<br>5.  Автоматизация:  База для создания автотестов. | 1.  Создать:  Кнопка "New" → Вкладка "Collection".<br>2.  Назвать:  Дать понятное имя (напр., "API Health Checks").<br>3.  Настроить:  (Опционально) добавить описание, pre-request scripts и tests для всей коллекции. |
|  3. Создание запросов в коллекциях  | Основной элемент — HTTP-запрос (GET, POST и т.д.), который сохраняется внутри коллекции. | 1.  Выполнение запросов:  Отправка и проверка API-эндпоинтов.<br>2.  Сохранение истории:  Все настройки запроса сохраняются для повторного использования.<br>3.  Тестирование:  К каждому запросу можно добавить скрипты-проверки. | 1.  Новый запрос:  Навести на коллекцию → "..." → "Add Request".<br>2.  Настроить:  <br>   -  Метод & URL:  `GET` `{{base_url}}/users`<br>   -  Params:  Query-параметры.<br>   -  Body:  Данные для POST/PUT (JSON, form-data).<br>   -  Headers:  `Content-Type: application/json`.<br>   -  Auth:  Настройки авторизации.<br>3.  Сохранить:  Дать имя и сохранить в коллекцию. |
|  4. Добавление фрагментов (Snippets)  | Предварительно написанные куски кода (на JavaScript) для быстрого добавления стандартной логики в тесты или pre-request скрипты. | 1.  Скорость:  Не писать код для типовых задач с нуля.<br>2.  Обучение:  Примеры для новичков.<br>3.  Стандартизация:  Команда использует одинаковые проверки. | 1.  Открыть вкладку:  В запросе перейти во вкладку "Tests" или "Pre-request Script".<br>2.  Выбрать фрагмент:  Справа панель "Snippets". Выбрать нужный (напр., "Status code: Code is 200").<br>3.  Добавить:  Нажать на фрагмент — код появится в редакторе. При необходимости адаптировать под себя. |
|  5. Запуск автотестов для коллекции  | Функция для запуска всех запросов в коллекции последовательно, выполняющая прикрепленные к ним скрипты (тесты). | 1.  Интеграционное тестирование:  Проверка целого сценария (напр., "создать юзера → получить его данные → удалить").<br>2.  Непрерывная интеграция (CI):  Запуск тестов в сборке (напр., через Newman).<br>3.  Регрессионное тестирование:  Быстрая проверка, что новые правки не сломали старое. | 1.  Написать тесты:  Для каждого запроса во вкладке "Tests" добавить скрипты (вручную или через фрагменты).<br>2.  Запустить коллекцию:  Открыть коллекцию → вкладка "Run".<br>3.  Настроить запуск:  Выбрать окружение, количество итераций.<br>4.  Старт:  Нажать "Run [Collection Name]".<br>5.  Анализ:  Посмотреть результаты в "Run Results". Зеленые галочки — тесты пройдены. |
