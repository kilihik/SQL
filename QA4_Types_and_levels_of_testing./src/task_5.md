# Чек-лист тестирования приложения "Найди и иди" Grey-box

##  Проверки работы с базой данных магазинов

### Структура и целостность данных
- Убедиться, что все обязательные поля магазинов заполнены (название, тип, координаты)
- Проверить корректность формата координат в базе данных
- Проверить, что типы магазинов соответствуют списку из ТЗ
- Убедиться в отсутствии дубликатов магазинов
- Проверить связи между таблицами (если есть связанные данные)

### Качество данных
- Проверить валидность географических координат всех магазинов
- Убедиться, что названия магазинов не содержат некорректных символов
- Проверить заполненность критически важных полей (адрес, часы работы)
- Убедиться в актуальности данных о магазинах

##  Проверки поисковых механизмов

### Поиск по названию
- Проверить поиск с разной длиной запроса (2 символа, 3+ символов)
- Убедиться, что поиск работает с разными регистрами букв
- Проверить поиск по части названия
- Убедиться, что поиск возвращает релевантные результаты
- Проверить ограничение на количество показываемых результатов

### Поиск по типу
- Проверить фильтрацию по каждому типу магазинов
- Убедиться, что поиск по типу возвращает все магазины этого типа
- Проверить комбинированный поиск (название + тип)
- Убедиться в корректности определения "ближайшего" магазина

## Проверки алгоритмов геолокации

### Расчет расстояний
- Проверить корректность расчета расстояния до магазинов
- Убедиться, что "ближайший" магазин действительно самый близкий
- Проверить работу с разными географическими точками
- Убедиться в точности отображения магазинов на карте

### Построение маршрутов
- Проверить построение маршрута до конкретного магазина
- Проверить построение маршрута до ближайшего магазина по типу
- Убедиться в корректности данных маршрута (дистанция, время)
- Проверить обработку различных стартовых точек

## Проверки сохранения данных

### История поиска
- Убедиться, что поисковые запросы сохраняются в базе
- Проверить ограничение на глубину истории поиска
- Убедиться в корректности отображения истории
- Проверить очистку истории поиска

### Сохранение состояния
- Проверить сохранение незавершенных маршрутов
- Убедиться в корректном восстановлении состояния при перезапуске
- Проверить время хранения временных данных
- Убедиться в целостности сохраняемых данных

## Проверки производительности

### Скорость поиска
- Измерить время отклика при поиске по названию
- Измерить время отклика при поиске по типу
- Проверить производительность при большом количестве магазинов
- Убедиться в отсутствии долгих SQL-запросов

### Оптимизация запросов
- Проверить использование индексов в базе данных
- Убедиться в эффективности поисковых запросов
- Проверить нагрузку на базу данных при активном использовании
- Убедиться в отсутствии утечек памяти

## Проверки обработки ошибок

### Ошибки базы данных
- Проверить поведение при недоступности базы данных
- Убедиться в корректной обработке ошибок запросов
- Проверить обработку некорректных данных из базы
- Убедиться в наличии понятных сообщений об ошибках

### Ошибки геолокации
- Проверить работу при отсутствии геолокации
- Убедиться в корректной работе с неточными координатами
- Проверить обработку ошибок картографического сервиса
- Убедиться в работе при ограниченной точности геолокации

## Проверки безопасности

### Защита данных
- Проверить защиту от SQL-инъекций в поиске
- Убедиться в безопасном хранении пользовательских данных
- Проверить валидацию входных параметров
- Убедиться в отсутствии утечки чувствительной информации

### Доступ к данным
- Проверить права доступа приложения к базе данных
- Убедиться в минимально необходимых привилегиях
- Проверить аудит операций с базой данных
- Убедиться в защите конфиденциальной информации магазинов

##  Интеграционные проверки

### Внешние сервисы
- Проверить интеграцию с картографическим сервисом
- Убедиться в корректной передаче координат
- Проверить обработку сбоев внешних сервисов
- Убедиться в согласованности данных между системами

### Системные интеграции
- Проверить работу с системой геолокации устройства
- Убедиться в корректном определении уровня заряда батареи
- Проверить управление точностью геолокации
- Убедиться в корректной работе с разрешениями

## Проверки с различными сценариями данных

### Разные регионы
- Проверить работу в густонаселенных районах
- Проверить работу в районах с малым количеством магазинов
- Убедиться в работе в разных часовых поясах
- Проверить обработку международных символов в названиях

### Граничные случаи
- Проверить поиск при отсутствии магазинов нужного типа
- Убедиться в работе при одинаковом расстоянии до магазинов
- Проверить обработку магазинов с одинаковыми названиями
- Убедиться в работе при экстремальных координатах
